## Description

`ColorShift` dissolves two scenes while a flat-color overlay arcs up to `overlayOpacityPeak` at the midpoint and recedes to transparent. The overlay functions as a tonal signal: `warm` uses the canonical amber accent (`#f59e0b`) to signal narrative warmth or time-of-day shifts; `cool` uses a restrained blue for contrast-scene cuts; `neutral` uses a near-white screen blend for editorial whiteout moments.

Unlike a flash cut, the overlay's opacity arc is gradual — viewers read it as a mood shift rather than a technical flash. The cross-dissolve underneath ensures no hard edge, making this the most editorial-light transition in the set.

```bash

```

The component lands in `remotion/transitions/color-shift.tsx` and is yours to edit.

## Usage

```tsx
import { AbsoluteFill, Composition } from "remotion";
import { ColorShift } from "@/remotion/transitions/color-shift";

function SceneA() {
  return <AbsoluteFill style={{ background: "#0a0a0a" }} />;
}

function SceneB() {
  return <AbsoluteFill style={{ background: "#141414" }} />;
}

function MyComposition() {
  return (
    <ColorShift
      direction="warm"
      durationFrames={24}
      overlayOpacityPeak={0.45}
      accentColor="#f59e0b"
      easing="literary"
      transitionStart={36}
      from={<SceneA />}
      to={<SceneB />}
    />
  );
}

export function Root() {
  return (
    <Composition
      id="MyColorShift"
      component={MyComposition}
      durationInFrames={90}
      fps={30}
      width={1920}
      height={1080}
    />
  );
}
```

<Warning title="BRAND.md amber restraint">
  `direction="warm"` introduces an amber overlay across the full composition width. Per BRAND.md, this counts as the accent usage for every frame it occupies. Avoid combining it with scenes that already contain amber title lines, accent borders, or icon fills — the overlay will flood the frame with competing warmth.
</Warning>
