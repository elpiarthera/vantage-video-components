## The steps timeline

`Button` accepts a `steps` prop — an array of `Step<ButtonAction>` objects that script the button's interaction. The full `ButtonAction` union is:

```ts
type ButtonAction =
  | "hover"    // lift + background crossfade
  | "press"    // scale down + darken
  | "release"  // return to hover level
  | "loading"  // label fades out, spinner fades in
  | "success"  // spinner fades out, checkmark draws
  | "reset"    // destructive flash, then return to idle
```

A realistic happy-path interaction — hover, click, load, succeed:

```tsx
import { Button } from "@/components/remocn/button";

export const Scene = () => (
  <Button
    label="Continue"
    steps={[
      { at: 10, action: "hover" },
      { at: 25, action: "press" },
      { at: 35, action: "loading" },
      { at: 70, action: "success" },
    ]}
  />
);
```

Each `at` value is a **Sequence-local authored frame**. If the component sits inside a `<Sequence from={30}>`, Remotion shifts the playhead automatically — `at: 10` always means "10 frames into this Sequence." See the [ui tier intro](/docs/ui) for how `useTimelineState` resolves the fold.

### Single-action shortcut

For a quick loading spinner or a one-step hover, pass a single step:

```tsx
// Spinner only — no label
<Button label="Save" steps={[{ at: 0, action: "loading" }]} />

// Hover state held for the full duration
<Button label="Sign up" steps={[{ at: 5, action: "hover" }]} />
```

### Using a preset scenario

The `scenario` prop is a preview-convenience alias for a fixed `steps` array. It exists so the live preview customizer can switch between canned interactions without the user manually editing step objects. In production compositions, prefer explicit `steps`.

| Scenario | Equivalent steps |
|---|---|
| `"happy"` | hover→press→loading→success |
| `"loading"` | loading (at 10) |
| `"error"` | hover→press→loading→reset |
| `"idle"` | `[]` (static button) |

```tsx
// Fine for quick previews; prefer steps={[...]} in real compositions
<Button scenario="happy" />
```

When both `steps` and `scenario` are supplied, `steps` wins.
